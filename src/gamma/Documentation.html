Gamma Interface Documentation

Module: gamma.interface

This module provides the primary user-facing API for the Gamma GPU-accelerated FEM solver. It abstracts the underlying CuPy kernels and domain management into a high-level FeaModel class, handling simulation initialization, time-stepping, and data orchestration (Zarr and VTK I/O).

Class: FeaModel

The core simulation manager. It bridges the physical geometry (domain_mgr), the numerical solver (heat_solve_mgr), and the output recording pipelines.

Initialization

__init__(self, input_data_dir, geom_dir, laserpowerfile, timestep_override=-1, 
         VtkOutputStep=1, ZarrOutputStep=0.02, outputVtkFiles=True, 
         verbose=True, CalcNodeSurfDist=False)

Initializes the simulation environment, loads the finite element mesh, and sets up the output directories.

Parameters

  • input_data_dir (str)
    Root directory containing the geometry and laser input data.
  • geom_dir (str)
    Sub-directory containing the specific .k mesh file and toolpath.crs file.
  • laserpowerfile (str | None)
    Filename (without .csv extension) of the laser power sequence. If None, the user must provide power manually during .step().
  • timestep_override (float, default=-1)
    Forces a specific Δt for the simulation. If < 0, it defaults to the temporal resolution of the laser toolpath.
  • VtkOutputStep (int, default=1)
    The frequency (in simulation micro-steps) at which 3D .vtk files are saved for visualization.
  • ZarrOutputStep (float, default=0.02)
    The frequency (in simulation micro-steps) at which numerical data is written to the Zarr store.
  • outputVtkFiles (bool, default=True)
    Master toggle for .vtk file generation. Set to False during optimization loops to preserve I/O bandwidth.
  • verbose (bool, default=True)
    Toggles terminal output for simulation progress and ETA.
  • CalcNodeSurfDist (bool, default=False)
    If True, initializes the simulation in "Auxiliary Mode" to calculate spatial relationships (node-to-laser and node-to-boundary distances) rather than thermal physics.

Core Execution Methods

run(self)

Executes the simulation from t=0 to the end of the predefined toolpath time (end_sim_time). It automatically steps through the physics solver and handles all I/O operations at the frequencies defined during initialization.

step(self, laser_power=None)

Advances the simulation by a single time step (Δt).

  • laser_power (float, optional)
    Dynamically overrides the laser power for the current step. If None, the solver reads the value from the loaded laser_power_seq.

Note: This method internally triggers the Zarr and VTK writing logic and forces GPU memory garbage collection (mempool.free_all_blocks()) at each Zarr output interval.

calc_geom_params(self)

A specialized execution loop that bypasses the thermal physics integration. Instead, it advances the laser along the toolpath purely to calculate and record spatial features:

  1. Node Deposition Timestamp: When an element becomes "active".
  2. Laser Distance: Nodal distance to the active heat source.
  3. Boundary Distance: Nodal distance to the closest free surface.

I/O & Cloud Methods

OneDriveUpload(self, rclone_stream, destination, BashLoc)

Automates the archiving and offloading of simulation results to cloud storage.

Action: Compresses the .zarr and .vtk output directories using tar, uploads them via rclone, and instantly deletes the local copies to free up the host drive.

RecordTempsZarr(self, active_nodes, active_nodes_prev, outputmode="structured")

Appends the current numerical state (temperatures, active elements, laser position) from the GPU to the Zarr store.

Performance Note: This function currently uses the zarr.append() method. For large-scale batch optimization, this method introduces an I/O bottleneck due to repeated disk re-sizing.

Data Recording Classes

The interface utilizes two helper classes to manage the Zarr storage schemas.

DataRecorder

Initializes the primary .zarr store for thermal physics outputs. It creates a structured hierarchy tracking the following streams at every ZarrOutputStep:

  • timestamp
  • dt_pos_x, dt_pos_y, dt_pos_z (Laser coordinates)
  • dt_laser_power
  • ff_dt_temperature (Full-field nodal temperatures)
  • ff_dt_active_elements / ff_dt_active_nodes (Element birth status)

AuxDataRecorder

Initializes a separate .zarr store (suffixed with _aux) utilized strictly by the calc_geom_params() method. It tracks:

  • ff_dt_dist_node_laser
  • ff_dt_dist_node_boundary
  • ff_timestamp_node_deposition
